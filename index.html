⚙️ First-time Setup

Create a Google Cloud project, enable Drive API + Sheets API, create an OAuth 2.0 Web Client ID at console.cloud.google.com, add this URL as an authorized origin. Then create a blank Google Sheet.

📘 How to Get Your Page Access Token

A Page Access Token lets the Apps Script post to Instagram and Facebook on your behalf. Takes about 10 minutes to set up once.

  1. Go to developers.facebook.com/apps → Create App → Other → Business
  2. Inside your app → Add Product → add Facebook Login for Business
  3. Go to Graph API Explorer
  4. Select your app → click Generate Access Token → add permissions:
    pages_manage_posts pages_read_engagement pages_show_list instagram_basic instagram_content_publish
  5. Click Generate Token → authorize → copy the token
  6. Paste it in Settings → Meta section → click Auto-fetch IDs → Save
⚠️ Explorer tokens expire in ~60 days. For a never-expiring token, use a System User token under Business Settings in your Meta developer app.

🐙 GitHub Actions Setup

The auto-merge runs on GitHub Actions free tier. Follow these steps once.

  1. Create a free GitHub account → new private repo
  2. Upload merge.py and .github/workflows/merge.yml
  3. Google Cloud → IAM & Admin → Service Accounts → Create → download JSON key
  4. Share your Sheet + both Drive folders with the service account email (Editor access)
  5. GitHub repo → Settings → Secrets → Actions → add:
      • SHEET_ID — your Google Sheet ID
      • GOOGLE_CREDS — entire contents of the JSON key file
  6. Actions tab → run Auto Merge & Schedule manually once to test
⚠️ GitHub Actions free tier: 2,000 min/month. Each merge run takes ~5-10 min = 200+ runs/month. More than enough for daily posting.
